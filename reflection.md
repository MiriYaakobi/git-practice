# מתי נשתמש בפקודות Git בעבודה אמיתית

* **`git stash`**: כשאני באמצע פיתוח פיצ'ר (קוד לא גמור) וצריכה לקפוץ לענף אחר כדי לבדוק משהו או לתקן באג, בלי ללכלך את ההיסטוריה בקומיט שבור.
* **`git commit --amend`**: כשיצרתי קומיט הרגע, אבל גיליתי ששכחתי להוסיף קובץ קטן או כשהייתה לי שגיאת כתיב בהודעת הקומיט.
* **`git rebase -i`**: כשאני עובדת על ענף פרטי שלי ורוצה לנקות ולסדר את היסטוריית הקומיטים (למשל למעוך קומיטים של טיוטה עם squash) לפני פתיחת ה-PR.
* **`git push --force-with-lease`**: כשאני חייבת לדרוס את ההיסטוריה בשרת (למשל אחרי שעשיתי Rebase מקומי), אבל רוצה רשת ביטחון שתוודא שאני לא מוחקת בטעות קוד שמישהו אחר דחף בינתיים.
* **`git cherry-pick`**: כשמתכנת אחר בצוות עשה תיקון בענף שלו, ואני צריכה לקחת בדיוק את הקומיט הבודד הזה לענף שלי, בלי למזג איתו את כל שאר העבודה הלא גמורה שלו.
* **`git reflog`**: כשעשיתי טעות (כמו מחיקת ענף או reset --hard שמחק לי עבודה) ואני צריכה לחזור אחורה בזמן ל-Hash שנעלם כדי להציל את הקוד.
* **`git bisect`**: כשיש באג במערכת ואין לי מושג מתי ואיפה הוא נוצר, ואני צריכה לאתר את הקומיט שזה קרה בו מהר (חיפוש בינארי במקום מעבר ידני).
* **`git branch -c <name>`**: כשמתחילים לעבוד על משימה חדשה וצריכים סביבת עבודה מבודדת כדי לא לשבור שום דבר בענף הייצור המרכזי.
